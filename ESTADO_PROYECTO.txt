ESTADO ACTUAL DEL PROYECTO - QTENGO

--- RESUMEN GENERAL ---
El proyecto es una aplicación de gestión empresarial (ERP/Pyme) desarrollada en Android con Jetpack Compose y Clean Architecture (parcial). Utiliza Room para la persistencia local y ViewModel con LiveData para la gestión del estado de la UI.

--- ESTRUCTURA DE MÓDULOS (LÓGICA) ---

1. Módulo de Inventario (Pyme/Product):
   - Frontend: ProductScreen.kt, ProductViewModel.kt.
   - Backend (Local): ProductDao.kt, Product.kt (Modelo).
   - Funcionalidad: Gestión de stock, alertas de stock bajo, categorías y unidades de medida.

2. Módulo de Tareas (Pyme/Task):
   - Frontend: TaskScreen.kt, TaskViewModel.kt.
   - Backend (Local): TaskDao.kt, Task.kt (Modelo).
   - Funcionalidad: Agenda de tareas, estados de completado, niveles de prioridad.

3. Módulo de Finanzas (Pyme/Finance):
   - Frontend: PymeFinanceScreen.kt, FinanceViewModel.kt.
   - Backend (Local): FinanceDao.kt, ExpenseDao.kt.
   - Funcionalidad: Registro de ingresos y gastos, balances financieros.

4. Módulo de Empleados (Pyme/Employee):
   - Frontend: EmployeeScreen.kt, EmployeeViewModel.kt.
   - Backend (Local): EmployeeDao.kt.
   - Funcionalidad: Gestión de personal.

5. Módulo de Proveedores (Pyme/Supplier):
   - Frontend: SupplierScreen.kt, SupplierViewModel.kt.
   - Backend (Local): SupplierDao.kt.
   - Funcionalidad: Directorio de proveedores.

--- CÓMO MODIFICAR LOS MÓDULOS ---

- Pantallas (UI): Se encuentran en `com.example.qtengo.pyme.ui`. Para cambiar el diseño, edita los archivos `...Screen.kt`.
- Lógica de Negocio (Frontend): Los `...ViewModel.kt` en la misma carpeta manejan cómo los datos se presentan a la pantalla.
- Base de Datos (Backend Local):
    - DAOs: En `com.example.qtengo.core.data.database.dao`. Aquí defines las consultas SQL (Insert, Update, Delete, Query).
    - Modelos: En `com.example.qtengo.core.domain.models`. Aquí defines qué datos tiene cada entidad (ej. un Producto tiene nombre, cantidad, etc.).
    - Database Config: `AppDatabase.kt` centraliza todas las tablas.

--- FRONTEND VS BACKEND ---
- Frontend: Jetpack Compose (UI) + ViewModels (Estado). Todo lo que el usuario ve y toca.
- Backend: Room Database (Local) + DAOs. Es el motor que guarda y recupera la información de forma persistente.
